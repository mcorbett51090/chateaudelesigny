# G5 — Red-team (failure modes of the SYNTHESIZED build-now plan)

Real, reproducible modes specific to *this* approach (not the input plans). Each has a trigger + mitigation.

| # | Failure mode | Trigger / repro | Sev | Mitigation (in plan) |
|---|---|---|---|---|
| R1 | **Hotlinked existing-site images break or are low-res.** Pulling château images by URL → 403/hotlink-block, or they're too small for full-bleed heroes → pixelated = anti-luxury. | Reference live-site image URLs directly; site adds hotlink protection or images are <1200px. | High | **Download** representative images into `src/assets/` at build (don't hotlink); apply duotone + grain + intentional crop so resolution limits read as art direction, not as low-quality; reserve a clearly-labeled swappable slot for owner's real hi-res. |
| R2 | **Duotone unification looks gimmicky / dated** if applied globally and heavy-handed. | Every photo forced to one heavy duotone → looks like a 2015 filter, not luxury. | Med | Use a **restrained** treatment: subtle warm monochrome on *secondary* imagery only, full-color on a few hero "money shots"; tune via CSS `mix-blend`/`filter` tokens so it's one-line adjustable; keep an escape hatch to full color per image. |
| R3 | **Agent-generated SVG crest/schematic looks amateur**, undercutting the high-end goal. | Naive auto-drawn crest with clumsy paths. | Med | Keep marks **typographic + geometric** (a refined monogram lockup + a clean architectural line-schematic), not illustrative heraldry; if it can't reach quality, fall back to an elegant wordmark — never ship a weak crest. |
| R4 | **Mailto-default inquiry form feels broken/cheap** on devices with no mail client, or pre-fills a huge ugly body. | User on a device without a configured mail handler clicks submit → nothing happens. | High | Adapter is **graceful**: primary path is a clean `mailto:` with a tidy templated body **and** the form always shows the direct email + phone prominently as the real CTA; detect no-handler and show a copy-to-clipboard fallback. The form is an enhancement, not the only path. |
| R5 | **Astro i18n routing pitfalls** — default-locale prefix decisions, broken language-switch links that don't preserve the current path, missing/incorrect `hreflang`. | Switch FR→EN on a deep page → 404 or returns home. | Med | Centralize locale logic in one helper (`getRelativeLocaleUrl`); switcher computes the equivalent path; a build-time check asserts every page exists in both locales + valid hreflang; test deep-page switching. |
| R6 | **Performance budget blown** by full-bleed hero media + galleries on mobile. | Large hero image/video + PhotoSwipe assets → LCP > 3s on 4G. | Med | Hard perf budget in CI-style check; responsive `<Image>` (AVIF/WebP, srcset), lazy galleries (dynamic import), **no autoplay video by default** (poster-first), preconnect/subset fonts. |
| R7 | **Placeholder legal/contact facts ship as if real**, creating a false or non-compliant Mentions légales. | Lorem/placeholder entity name left in legal page at "done". | Med | All user-held facts live in ONE `site.config`/content file with **explicit `TODO:` markers**; a build-time lint fails the "done" check if any `TODO:` legal/contact placeholder remains; surface the must-provide list at handoff. |
| R8 | **Scope creep re-imports the cut features** (real 360, tile map, hosted CMS) mid-build and stalls on accounts. | Tempting to "just wire Resend quickly" → blocked on DNS. | Low-Med | Architectural rule: cut features exist ONLY as swappable slots/adapters; wiring a live provider is an explicit post-ship upgrade task, never in the build-now critical path. |

**Verdict:** No unmitigated high-severity blocker. R1 and R4 are the two high-severity modes and both have concrete, in-plan mitigations (download+art-direct images; graceful mailto adapter with always-visible direct contact). Cleared to synthesize. No loop-back to G2/G4 required.
